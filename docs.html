FaRl Language
Reference

FaRl (Financial aRrangement Language) is the scripting language built into Corren. It is designed to express financial flows clearly, safely, and with compile-time validation. Every script is atomic — if any statement fails, nothing is committed.

Installation

Requires Go 1.16+. Build from source:

bash
git clone https://github.com/amezianechayer/corren
cd corren && go build -o corren .
sudo mv corren /usr/local/bin/
corren --help

Quick Start

bash
# Terminal 1
corren server start

# Terminal 2
cat > script.farl << EOF
transfer [DZD.2 10000] (
  from @world
  to   @banque:reserve
)
EOF
corren exec quickstart script.farl
200 {"ok":true}

curl http://localhost:3068/quickstart/accounts/@banque:reserve

Overview

A FaRl script is a sequence of statements separated by newlines. Each statement either moves money, prints a value, or sets metadata. The entire script executes as a single atomic transaction.

Compile-time safety

All errors are caught before execution. Insufficient balances, type mismatches, allocation overflows, and invalid source chains are detected at compile time. No partial states, ever.

farl
# Variables declared at the top
var $montant: monetary
var $dest: account

# Statement 1
transfer $montant (
  from @world
  to   $dest
)

# Statement 2 — uses balance from statement 1
transfer [DZD.2 *] (
  from $dest
  to   @archive
)

Monetary Syntax

The monetary type has two forms depending on whether you need decimal precision:

farl
# With precision — [ASSET.decimals amount]
[DZD.2 5000]      # Algerian Dinar, 2 decimal places
[EUR.2 10000]    # Euro, 2 decimal places
[USD.2 10000]    # US Dollar, 2 decimal places
[TND.3 5000]     # Tunisian Dinar, 3 decimal places
[NGN.2 5000]     # Nigerian Naira, 2 decimal places

# Without precision — [ASSET amount]
[COIN 100]        # COIN token, no decimals
[CREDIT 50]      # CREDIT token

# Transfer all balance
[DZD.2 *]         # all DZD.2 balance
[COIN *]          # all COIN balance
Precision

The precision number (e.g. .2 in DZD.2) is part of the asset name — it is not used for arithmetic. DZD.2 and DZD are two different assets. Amounts are always stored as integers.

Variables

Variables are declared at the top of the script before any statements. They are injected as JSON when executing via the API.

farl
# Simple declaration
var $name: type

# With metadata origin
var $name: type = meta($account, "key")

transfer

The transfer statement moves a monetary amount from a source to a destination.

farl — syntax
transfer <monetary | monetaryAll> (
  from <source>
  to   <destination>
)

The from and to clauses can be in any order:

farl
# Both orderings are valid
transfer [DZD.2 1000] (
  from @alice
  to   @bob
)

transfer [DZD.2 1000] (
  to   @bob
  from @alice
)
Balance tracking

Corren tracks balance changes within the script. A transfer can spend funds received by a previous transfer in the same script, even if the initial database balance was 0.

Sources

Single account

farl
from @alice
from $vendeur    # variable

Ordered cascade (in-order)

Drains each source in order until the target amount is reached:

farl
from {
  @wallet:ameziane    # drained first
  @credit:ameziane    # then this
  @world              # fallback (unlimited)
}

Capped source (max)

Take at most a fixed amount, excess is returned:

farl
max [DZD.2 1000] from @alice

# Can be combined with cascade
from {
  max [DZD.2 500] from @alice
  @bob
}

Allotment source

Take a proportional amount from each source:

farl
from {
  60%       from @alice
  35.5%     from @bob
  4.5%      from @charlie
}
Restriction

You cannot use * (transfer all) with an allotment source. The total must be a fixed amount.

Destinations

Single account

farl
to @bob
to $vendeur    # variable

Allotment destination

farl
to {
  85/100    to @vendeur:yanis
  10/100    to @plateforme:frais
  5/100     to @taxes
}

With remaining

farl
to {
  10%       to @plateforme
  5%        to @taxes
  remaining  to @vendeur    # gets 85%
}

Ordered (in-order) destination

Fill each destination in order until its capacity is reached:

farl
to {
  max [DZD.2 100] to @bob   # capped at 100
  @charlie                   # gets the rest
}

Dynamic portion (variable)

farl
var $commission: portion

to {
  $commission  to @plateforme
  remaining    to @vendeur
}

Rounding

When dividing non-divisible amounts, Corren distributes the remainder to the first portion. The total always equals the original amount exactly.

@world

@world is a special built-in account representing money creation. It emits unlimited funds with no balance check — like a central bank.

farl
# ✅ Emit a fixed amount
transfer [DZD.2 10000] (
  from @world
  to   @banque:reserve
)

# ✅ Fallback in a cascade
from {
  @wallet
  @world
}

# ❌ Cannot transfer all from @world
transfer [DZD.2 *] ( from @world  to @bob )

# ❌ Cannot use @world in allotment source
from { 50% from @world  50% from @alice }

Metadata (read)

Variables can be resolved from account metadata at runtime using meta(account, "key"):

farl
var $sale:       account
var $seller:     account = meta($sale,   "seller")
var $commission: portion = meta($seller, "commission_rate")

transfer [DZD.2 1000] (
  from $sale
  to   {
    $commission  to @plateforme
    remaining    to $seller
  }
)

set metadata new

FaRl can write metadata on transactions and accounts using set statements.

Set transaction metadata

farl
# Single key
set transaction metadata "reference" = "PAY-2026-001"
set transaction metadata "amount_dzd" = 5000

# Block form
set transaction metadata {
  "reference" = "PAY-2026-001"
  "channel"   = "mobile"
  "region"    = "alger"
}

Set account metadata

farl
set account metadata of @vendeur:yanis key "commission_rate" = 10/100
set account metadata of @vendeur:yanis key "tier"            = "gold"
Atomicity

Metadata writes are part of the same atomic transaction. If a later statement fails, all set statements in the script are also rolled back.

print & fail

print

farl
print 42
print 10 + 5 - 2
print @alice

fail

Immediately aborts the script and rolls back all changes.

farl
fail

Example — Marketplace split

farl
transfer [DZD.2 5000] (
  from @client:ameziane
  to   @courses:0587:paiement
)

transfer [DZD.2 5000] (
  from @courses:0587:paiement
  to   {
    90/100  to @chauffeur:yanis
    10/100  to @plateforme:frais
  }
)

set transaction metadata "reference" = "COURSE-0587"

Example — Source cascade

farl
transfer [DZD.2 3000] (
  from {
    @wallet:ameziane    # balance: 1000 → used first
    @credit:ameziane    # balance: 2000 → used next
    @world              # fallback
  }
  to @commande:0089
)
# Result: 1000 from wallet + 2000 from credit = 3000

Example — Dynamic commission

farl
var $sale:   account
var $seller: account = meta($sale,   "seller")
var $rate:   portion = meta($seller, "commission_rate")

transfer [DZD.2 10000] (
  from $sale
  to   {
    $rate      to @plateforme:commission
    remaining  to $seller
  }
)

Example — Allotment source

farl
transfer [DZD.2 1000] (
  from {
    60%  from @alice     # contributes 600
    35%  from @bob       # contributes 350
    5%   from @charlie   # contributes 50
  }
  to @destinataire
)

Example — Set metadata

farl
# Set a seller's commission rate
set account metadata of @vendeur:yanis key "commission_rate" = 12/100

# Now use it in a transfer
var $seller: account                              # injected via vars
var $rate:   portion = meta($seller, "commission_rate")

transfer [DZD.2 5000] (
  from @world
  to   {
    $rate     to @plateforme
    remaining to $seller
  }
)

HTTP API — Execute a script

bash
curl -X POST http://localhost:3068/quickstart/script \
  -H "Content-Type: application/json" \
  -d '{
    "plain": "transfer [DZD.2 1000] (\n  from @world\n  to @alice\n)",
    "vars": {}
  }'
FieldTypeDescription
plainstringrequiredThe FaRl script content
varsobjectoptionalVariable values as JSON
