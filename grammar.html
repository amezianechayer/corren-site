Grammar Reference

FaRl Grammar

Complete ANTLR4 grammar reference for FaRl. This page documents every token and parser rule. The grammar is defined in FaRl.g4 and compiled using ANTLR 4.9.2.

Keywords

Reserved words that cannot be used as identifiers:

printfailtransfer fromtomax keepallvar settransactionmetadata ofkeymeta remaining

Type Tokens

Literal Tokens

TokenPatternExamples
ACCOUNT@ [a-z_] [a-z0-9_:]*@alice, @users:001, @world
ASSET[A-Z] [A-Z0-9]*DZD, EUR, USD, NGN, TND
NUMBER[0-9]+42, 5000, 0
VARIABLE_NAME$ [a-z_] [a-z0-9_]*$montant, $vendeur
RATIO[0-9]+ / [0-9]+85/100, 1/3
STRING" [a-zA-Z0-9_/.] * ""seller", "commission_rate" — no spaces or %
NEWLINE[\r\n]+Line separator — significant in grammar
Note

WHITESPACE (spaces and tabs) and comments (/* ... */ and // ...) are skipped by the lexer. NEWLINE is not skipped — it is used as a statement separator.

script

The top-level rule. A script is zero or more variable declarations followed by one or more statements.

scriptstart rule
script
    : NEWLINE*
      (varDecl NEWLINE+)*
      stmts+=statement (NEWLINE+ stmts+=statement)*
      NEWLINE? EOF
    ;

statement

Five statement types are supported:

statementparser rule
statement
    // 1. Print a value
    : PRINT expr=expression
        # Print

    // 2. Abort the script
    | FAIL
        # Fail

    // 3. Move money
    | TRANSFER (mon=expression | monAll=monetaryAll) LPAREN NEWLINE
        ( FROM src=valueAwareSource NEWLINE TO dest=destination
        | TO dest=destination NEWLINE FROM src=valueAwareSource) NEWLINE RPAREN
        # Transfer

    // 4. Set transaction metadata (single key)
    | SET TRANSACTION METADATA STRING EQ metadataValue
        # SetTxMeta

    // 5. Set transaction metadata (block)
    | SET TRANSACTION METADATA LBRACE NEWLINE
        (metadataEntry NEWLINE)+
      RBRACE
        # SetTxMetaBlock

    // 6. Set account metadata
    | SET TY_ACCOUNT METADATA OF expression KEY STRING EQ metadataValue
        # SetAccountMeta
    ;

varDecl

varDeclparser rule
varDecl
    : VAR name=VARIABLE_NAME COLON ty=type_ (EQ orig=origin)?
        # VarTyped
    ;

type_
    : TY_ACCOUNT | TY_ASSET | TY_NUMBER
    | TY_MONETARY | TY_PORTION | TY_STRING
    ;

expression

expressionparser rule
expression
    : lhs=expression op=(OP_ADD | OP_SUB) rhs=expression  # ExprAddSub
    | lit=literal                                          # ExprLiteral
    | variable=VARIABLE_NAME                               # ExprVariable
    ;

literal
    : ACCOUNT   # LitAccount
    | ASSET     # LitAsset
    | NUMBER    # LitNumber
    | monetary  # LitMonetary
    ;

monetary

monetaryparser rule
monetary
    // With precision: [DZD.2 5000]
    : LBRACK asset=ASSET DOT precision=NUMBER amount=NUMBER RBRACK  # MonetaryLit

    // Without precision: [COIN 100]
    | LBRACK asset=ASSET amount=NUMBER RBRACK                       # MonetaryNoPrecision
    ;

monetaryAll

monetaryAllparser rule
monetaryAll
    // Transfer all with precision: [DZD.2 *]
    : LBRACK asset=ASSET DOT precision=NUMBER STAR RBRACK  # MonetaryAllPrec

    // Transfer all without precision: [COIN *]
    | LBRACK asset=ASSET STAR RBRACK                       # MonetaryAllNoPrec
    ;

source

sourceparser rule
source
    : expression               # SrcAccount
    | sourceMaxed              # SrcMaxed
    | sourceInOrder            # SrcInOrder
    ;

valueAwareSource

Used in the from clause of a transfer. Extends source with allotment support:

valueAwareSourceparser rule
valueAwareSource
    : source          # Src
    | sourceAllotment # SrcAllotment
    ;

sourceInOrder

sourceInOrderparser rule
sourceInOrder
    : LBRACE NEWLINE (sources+=source NEWLINE)+ RBRACE
    ;

Example: { @alice\n @bob\n @world\n }

sourceMaxed

sourceMaxedparser rule
sourceMaxed
    : MAX max=expression FROM src=source
    ;

Example: max [DZD.2 1000] from @alice

sourceAllotment

sourceAllotmentparser rule
sourceAllotment
    : LBRACE NEWLINE
        (portions+=allotmentPortion FROM sources+=source NEWLINE)+
      RBRACE
    ;

destination

destinationparser rule
destination
    : expression               # DestAccount
    | destinationMaxed         # DestMaxed
    | destinationInOrder       # DestInOrder
    | destinationAllotment     # DestAllotment
    ;

destinationInOrder

destinationInOrderparser rule
destinationInOrder
    : LBRACE NEWLINE (dests+=destination NEWLINE)+ RBRACE
    ;

destinationMaxed

destinationMaxedparser rule
destinationMaxed
    : MAX max=expression TO dest=destination
    ;

Example: max [DZD.2 100] to @bob

destinationAllotment

destinationAllotmentparser rule
destinationAllotment
    : LBRACE NEWLINE
        (portions+=allotmentPortion TO dests+=destination NEWLINE)+
      RBRACE
    ;

allotmentPortion

allotmentPortionparser rule
allotmentPortion
    : RATIO                          # allotmentPortionConst        // 85/100
    | NUMBER ('.' NUMBER)? PERCENT  # allotmentPortionConstPercent  // 12.5%
    | por=VARIABLE_NAME             # allotmentPortionVar           // $rate
    | PORTION_REMAINING              # allotmentPortionRemaining     // remaining
    ;

origin

Used in variable declarations to resolve a value from account metadata:

originparser rule
origin
    : META LPAREN acc=expression ',' key=STRING RPAREN
    ;

// Example:
// var $seller: account = meta($sale, "seller")

metadataEntry & metadataValue

metadataEntry / metadataValueparser rule
metadataValue
    : expression  # MetaValueExpr   // any expression: number, string, account...
    | RATIO       # MetaValueRatio  // e.g. 1/8
    ;

metadataEntry
    : STRING EQ metadataValue
    ;

// Examples:
// "commission_rate" = "10%"
// "amount" = 5000
// "seller" = @users:001
