Questions fréquentes

Corren est-il open source ?
Le code source est disponible sur GitHub. Nous travaillons sur la licence officielle — le core FaRl et le moteur de ledger seront sous licence open source. Les modules entreprise (intégrations SATIM, PAPSS) resteront propriétaires.
Quelles bases de données sont supportées ?
Corren supporte SQLite (embarqué, idéal pour le développement) et PostgreSQL (production). Le driver est configurable dans corren.yaml. MySQL est prévu dans une prochaine version.
Est-ce que Corren gère les transactions ISO 8583 / SATIM ?
Le connecteur SATIM/CIB ISO 8583 est en cours de développement et prévu pour Q3 2026. Les clients Enterprise auront accès en priorité. En attendant, Corren s'intègre avec n'importe quel système via son API REST.
Peut-on déployer Corren en interne (on-premise) ?
Oui. Corren est un binaire Go unique sans dépendances externes. Il se déploie sur n'importe quel serveur Linux. Le plan Enterprise inclut un accompagnement pour les déploiements air-gappés et les environnements VPC.
Quelle est la différence avec une base de données classique ?
Corren n'est pas une base de données générique. C'est un ledger programmable : chaque opération est une transaction double-entrée atomique exprimée en FaRl. Il garantit l'intégrité financière par construction — pas de soldes négatifs inattendus, pas d'états partiels.
Corren supporte quelles devises ?
Toutes. Les actifs sont définis librement : DZD.2 (Dinar algérien), EUR.2, USD.2, TND.3, mais aussi COIN, CREDIT, POINT — n'importe quel identifiant en majuscules. La précision décimale est configurable par actif.
